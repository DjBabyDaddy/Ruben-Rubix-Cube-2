# RUBE Memory System Schema

## 1. Persistent Memory (`memory/memory.json`)

Loaded at boot by `memory_manager.py:load_memory()`. Updated after each interaction
if the LLM returns a `memory_update` key.

```json
{
  "identity": {
    "name": {"value": "Trell", "updated": "2026-03-01"}
  },
  "preferences": {
    "nicknames": {
      "value": {"mama": "5045551234", "dad": "5045559876"}
    },
    "emails": {
      "value": {"mama": "mama@example.com"}
    }
  },
  "relationships": {},
  "emotional_state": {}
}
```

### How Memory Gets Updated
The LLM can include `"memory_update": {"key": {"value": "..."}}` in its JSON response.
`main.py` calls `update_memory(mem_update)` which merges into `memory.json`.

---

## 2. Session Memory (`memory/temporary_memory.py`)

In-memory only (not persisted to disk). Resets each session.

### Tracked State
| Attribute | Type | Purpose |
|-----------|------|---------|
| `history` | list[dict] | Last 10 conversation turns |
| `last_user_text` | str | Most recent user input |
| `last_ai_response` | str | Most recent LLM response |
| `parameters` | dict | Dynamic state (geo location, etc.) |
| `_pending_actions` | list[dict] | Deferred compound commands |
| `_pending_edits` | list[dict] | File edit approval queue |
| `interaction_count` | int | Counter for periodic reminders |
| `last_search_query` | str | Cache for search dedup |
| `last_search_result` | str | Cache for search readout |

### Pending Edits Schema
```json
{
  "id": "a1b2c3d4",
  "file_path": "actions/web_search.py",
  "reason": "Fix timeout handling",
  "old_content": "...",
  "new_content": "...",
  "diff": "...",
  "source": "self_assessment",
  "timestamp": "2026-03-13T18:30:00",
  "status": "pending"
}
```

## 3. Feedback Log (`memory/feedback_log.json`)

Append-only log of every action execution. Max 500 entries (oldest rotated out).
Written by `feedback_logger.py:log_action_result()`.

### Entry Schema
```json
{
  "timestamp": "2026-03-13T18:30:00.123456",
  "intent": "search",
  "parameters": {"query": "saints score"},
  "success": true,
  "exec_time_ms": 2100.5,
  "error": ""
}
```

### Analysis Functions
- `get_recent_lessons(lookback=50)` — Summarizes failure patterns for LLM injection
- `generate_self_assessment()` — Returns up to 5 actionable insights:
  - High failure rate intents (>30% fail, >=3 failures)
  - Consistently slow actions (>10s, >=3 occurrences)
  - Repeated failure patterns (same intent+error >=3 times)

---

## 4. Pending Edits File (`memory/pending_edits.json`)

Persisted version of the approval queue. Each entry matches the schema above.
Read/written by `TemporaryMemory` methods.

## 5. Suggestions (`memory/suggestions.json`)

List of optimization suggestions generated by n8n analysis workflow.
Read at startup by `memory_manager.py:get_startup_suggestions()`.
